## Active Decisions

1. **Context injection is one-time + per-turn**: HANDOFF.md, STATE.txt, scratchpad, and git diff
   are injected once per session. Orchestration status, interjections, and progress checks run
   every turn.
2. **Safety gate pattern**: All file and shell operations go through ReadSafetyGate and
   WriteSafetyGate in core/safety.py, enforcing workspace boundaries.
3. **Sub-agents are isolated**: Each sub-agent gets its own context, turn budget, and
   broadcast inbox. Parallel execution via agents/agent_runtime.py with auto-extension for
   productive agents.
4. **Circuit breaker**: Detects repeated identical tool calls and warns before API cost
   spirals (core/context_inject.py, 6-call window, 3-repeat threshold).
5. **Memory is SQLite**: memory/memory.py uses SQLite for persistence with separate session DBs.
   Learnings are tagged and searchable.
6. **Injection flags reset in bootstrap.init_session()**: Context injection flags
   (_scratchpad_injected, _git_diff_injected, _handoff_injected, _state_txt_injected)
   are reset once per session in core/bootstrap.init_session(), NOT per turn in
   run_agent_turn(). This gives injectors one chance per session, by design.
7. **build_startup_context lives in prompt.py**: The startup context builder is defined
   in core/prompt.py and imported directly, not re-exported through config.py.
8. **Unified orphan stripping in memory_prune.py**: ``_strip_orphaned_tool_messages``
   is the single source of truth, living in ``memory/memory_prune.py``. Both ``api.py``
   and ``memory/memory.py`` import from there. ``truncate=False`` (default) for API calls
   removes individual orphans; ``truncate=True`` for persistence truncates at the first
   incomplete sequence. Backward-compat aliases kept in api.py.
9. **Content-aware compression everywhere**: ``context_inject._compress_stale_tool_results``
   delegates to ``memory_prune._compress_tool_results`` -- per-tool-type compression with
   knowledge of request parameters (offset/limit for read_file, match patterns for search_files).
10. **Cache caps**: ``_clean_messages_cache`` capped at 16 entries.
    Prevents unbounded growth from stale message-list entries.
11. **MemoryStore.save() decomposed**: _prepare_messages() handles cleaning, compression,
    pruning, and summarisation. _write_messages() handles SQL writes with retry logic.
    _maybe_vacuum() handles opportunistic DB compaction. save() is a two-line dispatch.
12. **AgentContext and reservations extracted**: ``tools/context.py`` holds AgentContext,
    _ContextProxy, _TOOL_CONTEXT, and set_context. ``tools/reservations.py`` holds the
    file reservation system. Both re-exported through ``tools/__init__.py`` for
    backward compatibility.
13. **Codebase map generation**: ``core/codebase_map.py`` uses AST (Python) and regex
    (JS/TS) to extract symbols, module groupings, and import dependencies. Injected
    into startup context as a ~2K token structural map, saving the agent 3-5
    exploratory tool calls at session start.
14. **Mid-session conversation compaction**: ``_compact_if_needed()`` in
    ``context_inject.py`` runs at the start of every turn. When the conversation
    exceeds 80% of the context window, oldest messages are pruned and summarized
    using the existing ``_prune_by_tokens`` / ``_summarize_pruned`` pipeline,
    preserving system prompt + recent context. Compaction targets 70% fill ratio
    to leave room for turn growth. (new 2026-06-06)
15. **Pre-edit risk briefing**: ``_inject_edit_risk_context()`` scans the most
    recent assistant message for ``edit_file`` / ``write_file`` tool calls and
    injects a risk briefing with: git modification frequency (hotspot detection),
    co-change analysis (files modified in same commits), and missing test
    coverage. Zero-token cost when no edits are pending. (new 2026-06-06)
16. **File-pattern conditional rules**: ``.mini_agent/rules.toml`` defines
    glob-pattern rules (e.g. ``core/*.py``, ``tests/*.py``). ``_inject_pattern_rules()``
    checks recent file operations and injects only the rules matching touched
    files. Each rule injected at most once per session. Zero-token when no
    matching files are being worked on. (new 2026-06-06)
17. **Task-to-file index**: ``TASKS.md`` provides a curated mapping from common
    development tasks (\"Add a new tool\", \"Modify system prompt\", \"Change
    memory storage\") to the relevant source files. Injected once at session
    startup alongside HANDOFF.md and STATE.txt. (new 2026-06-06)
18. **Tree-sitter parser**: ``core/tree_sitter_parser.py`` provides multi-language
    (Python, JS, TS) symbol extraction via tree-sitter queries, with regex/AST
    fallback. Used by codebase_map for more accurate symbol extraction and by
    knowledge_graph for call-graph construction. Lazy-loads language grammars.
19. **Knowledge graph**: ``core/knowledge_graph.py`` builds an entity-relationship
    graph (calls, imports, inherits, defines) from workspace source files.
    Exposes ``find_related()`` (neighborhood), ``trace_path()`` (shortest path BFS),
    ``get_subgraph()``, ``find_callers_of_file()``, and ``ensure_graph_built()``.
    Wired into edit risk briefing, modified-files checkpoint, and post-edit
    verification to surface caller impact before and after edits.
20. **Git blame in edit risk briefing**: ``_inject_edit_risk_context()`` now
    includes ``git blame --line-porcelain`` analysis, showing top authors by
    line count for each file about to be edited. Helps the agent understand
    ownership context before making changes.
21. **Post-edit verification injection**: ``_inject_post_edit_verification()``
    runs every 6 turns after the modified-files checkpoint, showing test
    coverage status for modified files and affected callers from the knowledge
    graph. Periodic reminder to verify changes haven't broken downstream code.
22. **Codebase map incremental cache**: ``core/codebase_map.py`` now maintains a
    cached symbol map (``_MAP_CACHE``) updated incrementally via
    ``update_file_in_map()`` on every file write/edit. Avoids full workspace
    re-scans. ``_extract_single_file()`` uses tree-sitter when available.

23. **Windows subprocess hardening (2026-06-11)**: On Windows, ``proc.communicate()`` and
    ``proc.wait()`` can hang forever in kernel I/O (antivirus hooks, filter drivers).
    ``tools/shell_ops.py`` now provides ``_communicate_windows()`` -- a shared helper that
    uses read threads (``_stream_reader``) + ``threading.Timer`` watchdog calling
    ``taskkill /F /T`` to kill the process tree when a timeout fires. Used by
    ``_run_shell``, ``_run_tests``, and ``_verify``.
    ``CREATE_NO_WINDOW`` prevents conhost.exe per-process console windows.
    ``CREATE_NEW_PROCESS_GROUP`` was REMOVED (caused first-call freeze on Windows
    11 with certain antivirus/security configs). ``taskkill /F /T /PID`` works
    without process groups.
    ``mini_agent_electron/main.js`` has a restart throttle (max 3 restarts in 30s with
    exponential backoff) and uses ``taskkill /F /T /PID`` on shutdown.

24. **Bash path quoting fix (2026-06-11)**: ``tools/shell_ops.py`` line 243 (was 241): the
    MSYS2 bash path ``C:\Program Files\Git\bin\bash.exe`` must be quoted when building
    the bash wrapper command. Changed from ``f'{bash} -c \"{command}\"'`` to
    ``f'\"{bash}\" -c \"{command}\"'``. Without quotes, the double-wrapping at lines 319-327
    caused bash to treat ``C:\Program`` as a command name. The double-wrapping block was
    also bypassed (``if _WINDOWS and False and _is_bash_available():``) to use cmd.exe
    ``shell=True`` directly, since the command is already wrapped in bash.

25. **Windows fork prep (2026-06-11)**: ``requirements.txt`` refreshed with Windows
    notes (PyTorch CPU-only option, Playwright separate install, Defender exclusions).
    Added ``pytest-timeout>=2.0`` for Windows test safety. Created ``WINDOWS_INSTALL.md``
    with comprehensive Windows 11 install guide including Python Store stub avoidance,
    Node.js 22+ requirement, Visual C++ redist, PATH length workarounds, and
    troubleshooting (Defender, Electron white screen, proxy config). README.md
    updated to reference the Windows guide.

26. **Direct file reads on Windows (2026-06-11)**: ``tools/file_ops.py`` line 340:
    ``_worker.py`` subprocess hangs at ``open()`` on some Windows 11 systems (possibly
    antivirus filter driver). Changed ``if _WINDOWS:`` to ``if False:`` so all reads
    use ``_read_file_direct()`` in-process. This avoids the subprocess hang but loses
    the protection against kernel-filter hangs that the subprocess was designed to
    provide. Pre-existing nested-quote issue: commands with ``\"`` break the bash
    wrapping at line 241.

27. **Startup warmup for Windows antivirus (2026-06-11)**: ``core/bootstrap.py``
    ``init_session()`` now runs ``cmd.exe /c rem`` at startup to absorb any first-call
    antivirus/EDR scan delay for cmd.exe and conhost.exe. Windows Defender and other
    EDR products can pause the first execution of a new binary for behavioral
    analysis, making the first ``run_shell`` tool call appear to \"freeze.\"
    The warmup runs during startup (before the first user prompt) so the delay
    is invisible to the user.

## Active Decisions (New -- ACI Upgrades 2026-06-11)

28. **Read-before-edit enforcement (ACI upgrade)**: ``tools/file_ops.py``: ``write_file``
    and ``_apply_single_edit`` now reject writes/edits to .py files that haven't been
    ``read_file``'d this session. The ``_READ_FILES`` set tracks all files read during
    a session. New file creation (non-existent files) is exempted. This prevents the
    agent from hallucinating edits to files it hasn't seen.

29. **Syntax validation gate (SWE-agent linter pattern)**: ``tools/file_ops.py``:
    ``_validate_python_syntax()`` uses ``compile()`` to check .py file content before
    any write/edit is applied. Catches syntax errors (missing colons, bad indentation,
    invalid assignment targets) BEFORE they're persisted to disk. Returns the exact
    SyntaxError with line number and pointer. Non-.py files are skipped.

30. **Explicit empty-output messages (SWE-agent ACI pattern)**: ``tools/shell_ops.py``:
    ``_run_shell`` now returns ``"Command completed successfully (no output)."`` when
    a command exits 0 with no stdout and no stderr. This eliminates the ambiguous
    empty-string result that could mean "nothing happened" vs "success with no output."

31. **Dangerous command detection**: ``tools/shell_ops.py``: ``_check_dangerous_command()``
    scans shell commands for 9 dangerous patterns (rm -rf, git push --force, sudo,
    chmod 777, dd, mkfs, raw disk redirect, format drive). Blocked by default;
    allowed only with explicit ``force=True``. Reduces accidental destructive actions.

32. **Search result overflow hint**: ``tools/shell_ops.py``: when search_files results
    hit the 200-result cap, the hint now says ``"There may be more matches. Narrow
    your search with a more specific pattern, a subdirectory path, or use find_symbol
    for symbol lookups."`` -- guiding the model toward precision (SWE-agent pattern).

33. **Per-result size budget**: ``memory/memory_prune.py``: ``_TOOL_RESULT_MAX_CHARS``
    (8000) now hard-truncates any individual tool result exceeding this size during
    compression. Prevents a single huge result (e.g., full project log) from consuming
    the context window. Truncated results include offset guidance.

34. **ACI prompt rules**: ``core/prompt.py``: new sections "Read-Before-Edit &
    Verify-After-Change (ACI guardrails)" and "Plan-before-Edit Enforcement" injected
    into the immutable system prompt. Covers all new behaviors: read-first, verify-after,
    file-scoped commands, empty-output meaning, search caps, dangerous commands, and
    plan-first workflow.

35. **Stronger post-edit verification**: ``core/context_inject.py``:
    ``_inject_post_edit_verification()`` now fires whenever new files have been modified
    since the last verification check, in addition to the existing 6-turn interval.
    This catches immediate post-edit verification needs without waiting for the
    periodic cycle.

36. **Knowledge Confidence Scale nudge (2026-06-12)**: ``core/context_inject.py``:
    ``_inject_confidence_web_search_nudge()`` monitors the conversation for patterns of
    low agent confidence and nudges it to use ``web_search``. Three triggers: (a) 3+
    consecutive codebase search misses (find_symbol/search_files returning nothing)
    with no successful results, (b) 2+ consecutive tool failures on edit/write/shell/test,
    (c) 6+ consecutive turns of read-only activity. Includes a 4-turn cooldown to avoid
    nagging. Constants: ``_CONFIDENCE_NO_RESULT_THRESHOLD=3``,
    ``_CONFIDENCE_FAILURE_THRESHOLD=2``, ``_CONFIDENCE_NUDGE_COOLDOWN_TURNS=4``.
    Tested in ``tests/test_confidence_nudge.py`` (15 tests).
    Bug fixes applied: (a) UnboundLocalError on malformed tool result JSON -- ``data`` now
    initialized before try/except. (b) ``break`` on productive turn killed failure/miss
    counting for earlier messages -- replaced with ``_stopped_read_only`` flag.

39. **Semantic response cache (2026-06-12)**: ``tools/semantic_cache.py``: new module
    implementing an in-memory semantic cache using the shared ``all-MiniLM-L6-v2``
    SentenceTransformer model (zero additional memory footprint). Caches non-tool-call
    LLM responses keyed by cosine similarity (threshold: 0.92) of the last user
    message embedding. Bounded to 128 entries with 1-hour TTL. Integrated into
    ``call_llm()`` in ``api.py`` -- cache lookup before API call, storage after
    successful plain-text response. Stats tracked on ``_TOOL_CONTEXT._semantic_cache_stats``.
    Cleared on session reset. Industry consensus: 15-25% cost reduction with zero
    quality risk.

40. **Multi-provider fallback chain (2026-06-12)**: ``api.py``: ``call_llm()`` now
    supports automatic failover to fallback providers on 429/5xx errors. Configured
    via ``ProviderDefaults.fallback_providers`` (tuple of provider names). DeepSeek
    defaults to ``("claude",)``. On primary failure, each fallback is tried
    sequentially with its own API key, URL, and model. Provider-specific params
    (``thinking``, ``reasoning_effort``, ``frequency_penalty``) are stripped from
    fallback payloads. Semaphore re-acquired for each fallback attempt. Only
    active for non-streaming calls. Improves effective availability from ~99.5%
    to ~99.95%.

41. **Retry timeout uses config constants (2026-06-12)**: ``retry.py``: ``_request_with_retry()``
    previously hardcoded ``timeout=(10, 120)`` while ``config.py`` defined
    ``HTTP_CONNECT_TIMEOUT=30`` and ``HTTP_READ_TIMEOUT=120`` as unused constants.
    Now imports and uses ``(HTTP_CONNECT_TIMEOUT, HTTP_READ_TIMEOUT)``, increasing
    connect timeout from 10s -> 30s. Same values as ``bootstrap.py``'s session-level
    timeout, so timeouts stay consistent whether or not a Session is passed.

42. **Frontend backend:response handler fix (2026-06-14)**: ``App.jsx`` had a bug where
    the ``backend:response`` event handler checked ``data.target === 'chat'``, but the
    Python backend NEVER sets a ``target`` field on response messages. This caused ALL
    response messages (slash-command output, model switch errors, etc.) to be silently
    dropped. Removed the guard — responses are now shown in the chat panel as
    ``msg-status`` lines. (Also affected ``/stats``, ``/session``, ``/workspace`` commands.)

## Known Issues
- `_json_rpc_shared.py` -- shared drain_stderr() utility used by lsp.py and mcp_client.py
- No semantic search across past sessions (current recall is session-only)
- Core packages (`core/`, `agents/`, `memory/`) use lazy `__getattr__` in `__init__.py` to
  break circular imports. Adding new public symbols requires updating the `__getattr__` chain.

42. **Hermes-style skill architecture (2026-06-14)**: ``tools/skills.py`` now implements
    the Hermes Agent skills pattern: skills are defined as SKILL.md files with YAML
    frontmatter + markdown body, discovered from disk (workspace ``skills/`` then
    ``~/.mini_agent/skills/``), with progressive disclosure (compact catalog at startup,
    full content injected on ``use_skill``). Supports ``skill_list()`` (cached catalog),
    ``skill_view(name)`` (full docs), ``get_active_skill_content()`` (injection-ready
    content for newly activated skills, once per session per skill), and
    ``reload_skills()`` (force re-discovery after writes). Backward-compatible
    ``SKILLS`` dict maintained via lazy init. ``_parse_frontmatter()`` is a zero-
    dependency YAML-like parser supporting inline lists, block lists, booleans, and
    comments. 10 SKILL.md files created in ``skills/``: git, test, lsp, web, agents,
    search, tasks, image, desktop, bootstrap. 25 new tests in
    ``tests/test_skills_hermes.py``.
