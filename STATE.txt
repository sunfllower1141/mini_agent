## Active Decisions

1. **Context injection is one-time + per-turn**: HANDOFF.md, STATE.txt, scratchpad, and git diff
   are injected once per session. Orchestration status, interjections, and progress checks run
   every turn.
2. **Safety gate pattern**: All file and shell operations go through ReadSafetyGate and
   WriteSafetyGate in core/safety.py, enforcing workspace boundaries.
3. **Sub-agents are isolated**: Each sub-agent gets its own context, turn budget, and
   broadcast inbox. Parallel execution via agents/agent_runtime.py with auto-extension for
   productive agents.
4. **Circuit breaker**: Detects repeated identical tool calls and warns before API cost
   spirals (core/context_inject.py, 6-call window, 3-repeat threshold).
5. **Memory is SQLite**: memory/memory.py uses SQLite for persistence with separate session DBs.
   learnings are tagged and searchable.
6. **Injection flags reset in bootstrap.init_session()**: Context injection flags
   (_scratchpad_injected, _git_diff_injected, _handoff_injected, _state_txt_injected)
   are reset once per session in core/bootstrap.init_session(), NOT per turn in
   run_agent_turn(). This gives injectors one chance per session, by design.
7. **build_startup_context lives in prompt.py**: The startup context builder is defined
  in core/prompt.py and imported directly, not re-exported through config.py.
8. **Unified orphan stripping in api.py**: ``_strip_orphaned_tool_messages(messages, *, truncate=False)``
  is the single source of truth for orphaned tool message cleanup.  ``truncate=False`` (default)
  for API calls removes individual orphans; ``truncate=True`` for persistence truncates at the
  first incomplete sequence.  Replaces 4 separate implementations across 3 files.
9. **Content-aware compression everywhere**: ``context_inject._compress_stale_tool_results``
  now delegates to ``memory_prune._compress_tool_results`` instead of doing lossy first-line-only
  compression.  The ``_original_content`` field is no longer used — compression is idempotent.
10. **Cache caps**: ``_clean_messages_cache`` capped at 16 entries, ``_complexity_cache`` at 32.
    Prevents unbounded growth from stale message-list entries while preserving incremental
    cleaning optimization across turns.

## Known Issues
- `_json_rpc_shared.py` — abandoned file, should adopt or remove
- `tools/__init__.py` — still too large (761 lines), could split further
- No semantic search across past sessions (current recall is session-only)
- Core packages (`core/`, `agents/`, `memory/`) use lazy `__init__.py` with `__getattr__` to
