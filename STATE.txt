# mini_agent — Architecture & State

A coding agent powered by DeepSeek V4 Pro. Terminal REPL + Textual TUI.
717 tests, 31 tools, SQLite memory.

## Key Architectural Decisions

- **api.py**: Extracted `call_deepseek` etc. to break circular import between llm.py ↔ sub_agent.py
- **.mini_agent.rules**: Project conventions injected into system prompt via `build_system_prompt()`
- **LSP**: 4 tools (definition, references, hover, diagnostics) — working with real pylsp. Fix: skip all header lines (Content-Type) before reading body.
- **Diff preview**: `DiffPreview` dataclass in safety.py; write_file/edit_file generate before writing; rendered in tui.py
- **Named constants**: All magic numbers extracted (`GIT_LOG_COUNT`, `SSE_PREFIX`, `TOOL_DETAIL_DISPLAY_LENGTH`, etc.)
- **Binary I/O**: LSP uses binary pipes with manual encode/decode (Content-Length counts bytes)
- **Mock paths**: Tests patch `api.*` or `sub_agent.*` directly (not `llm.*`)

## Previous Session (2025-07-17)

### Multi-Agent Communication Overhaul
- **Problem**: Communication plumbing existed (agent_message, agent_handoff, agent_inbox, agent_subscribe, agent_read) but prompts never told agents to use them. Orchestrator only polled; sub-agents worked silently.
- **Fix**: Injected inter-agent comms guidance into 3 places:
  1. `prompt.py` "Orchestrator mode" section — new "Inter-agent communication" block covering all 5 comm tools + heartbeat expectations + Push > Poll guidance
  2. `sub_agent.py` `_SUB_AGENT_SYSTEM_PROMPT` — sub-agents now told to broadcast progress, send heartbeats every ~3 turns, check inbox first each turn
  3. `sub_agent.py` `run_sub_agent()` — task_id injected as system message so sub-agents can use agent_inbox/agent_handoff with `target=`
- **agent_inbox fix**: Made `task_id` optional — defaults to caller's own ID. Parent orchestrator can now call `agent_inbox()` without knowing its task_id.
- **Loop-enforced nudges**: Every 3 turns, the sub-agent turn loop injects a COMMUNICATION NUDGE system message requiring a heartbeat + inbox check. This guarantees communication even if the LLM "forgets."
- **Test**: `test_prompt.py` limit raised 15,000 → 17,000. 717 passed, same 6 pre-existing failures.
